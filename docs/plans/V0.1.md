# Plan: V0.1 вЂ” Client-Server Comms

## Context

V0.1 establishes the minimal infrastructure to prove client-server communication works before any game logic is built. The goal is: create a room, navigate to the board, have players join from their phones, and send a ping from a player screen to the board. No game mechanics, no Phaser вЂ” just wiring up Colyseus, routing, and a proof-of-concept message flow.

Relevant shaping doc: `docs/shaping/V0.1 - client-server comms.md`

## Slices

### Slice 0 вЂ” Repo Scaffold
**Goal**: both server and client start without errors.

- Root `package.json`: `workspaces: ["server", "client"]`, `"dev": "concurrently \"npm run dev -w server\" \"npm run dev -w client\""`
- `server/package.json`: deps `colyseus`, `@colyseus/express`, `express`; script `"dev": "node --watch index.js"`
- `server/index.js`: Express + Colyseus boilerplate, listens on `process.env.PORT || 2567`
- `client/package.json`: dep `vite`; scripts `dev`, `build`
- `client/vite.config.js`: proxy `/colyseus` в†’ `http://localhost:2567` in dev
- `client/index.html` + `client/src/main.js`: renders "Mario Cartes" text
- `.gitignore`

**Verify**: `npm install && npm run dev` в†’ server on 2567, Vite on 5173, no errors.

---

### Slice 1 вЂ” Game Room Creation + Board Screen
**Goal**: home page creates a room and navigates to the board; board shows connected players live.

- `server/index.js`: add `GET /create` в†’ `gameServer.createRoom("game", {})`, return `{ id }`
- `server/rooms/GameRoom.js`:
  - `onCreate`: store `this.roomId`
  - `onJoin(client, options)`: record `{ sessionId, type: options.type }`
  - `onLeave`: remove client from list
  - Broadcast `"players"` state after each join/leave
- `client/src/home.js`: button в†’ `fetch("/create")` в†’ `window.location = /game/:id/board`
- `client/src/board.js`: connect to Colyseus room as `{ type: "board" }`; on `"players"` message, render list of connected player sessions
- `client/src/main.js`: route `/` в†’ home, `/game/:id/board` в†’ board

**Verify**: click "Create game" в†’ navigates to board в†’ board shows "(board) connected".

---

### Slice 2 вЂ” Player Join + Ping
**Goal**: a player navigating to `/game/:id/player` joins the room; pressing "Ping" shows a message on the board for 1 second.

- `server/rooms/GameRoom.js`: `onMessage("ping", (client) => this.broadcast("ping", { from: client.sessionId }))`
- `client/src/player.js`: connect as `{ type: "player" }`; render a single "Ping" button that sends `room.send("ping")`
- `client/src/board.js`: on `"ping"` message, show `"ping from [sessionId]"` div, remove after 1000ms
- `client/src/main.js`: route `/game/:id/player` в†’ player

**Verify**: open board in one tab, open player in another в†’ player list updates; press Ping в†’ board shows ping message for 1 second.
