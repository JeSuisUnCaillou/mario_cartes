# v0.14 - Win condition and start over

## Context

The game currently has no end condition — players race around a 14-cell circular track indefinitely. This feature adds lap counting (3 laps), a ranking system, a leaderboard, and a "start over" button to reset the game.

## Slice 1: Server — Lap tracking and win detection

**Files**: `server/rooms/GameRoom.js`, `server/tests/lobby.test.js`

### Server changes (`GameRoom.js`)

1. **Add `lapCount` to player state** (in `onJoin` player creation):
   - `lapCount: 1` (starts at lap 1)

2. **Add `ranking` to room state** (in `onCreate`):
   - `this.ranking = []` (array of playerIds in finish order)

3. **Mark finish line in track data** (`assets/racetrack_0_cells.json`):
   - Add `"finish_line": true` property to cell 1

4. **Detect lap completion in movement loop** (inside `playCard` handler):
   - After moving to next cell, check if `this.cells.get(player.cellId).finish_line` is truthy
   - If so: increment `player.lapCount`
   - If `player.lapCount > 3`: player finished → add to `this.ranking`, broadcast `"playerFinished"`
   - After movement loop: if `this.ranking.length === this.players.size` → phase = "finished", broadcast `"gameFinished"`

5. **Skip finished players in `_advanceTurn()`**:
   - Skip players in `this.ranking` (same as disconnected skip)

6. **Include lap/ranking in broadcasts**:
   - `broadcastPlayers()`: add `lapCount` and `finished`
   - `broadcastGameState()`: add `ranking`

7. **Handle finished player's turn end**: Auto-end turn, discard hand, don't draw new cards, advance turn.

### Tests

- Player completes a lap: lapCount increments
- Player completes 3 laps: added to ranking
- Finished player is skipped in turn order
- All players finish: phase becomes "finished"
- `broadcastPlayers` includes `lapCount` and `finished`

## Slice 2: Server — Start over

**Files**: `server/rooms/GameRoom.js`, `server/tests/lobby.test.js`

### Server changes

1. **Add `"startOver"` message handler**:
   - Only allowed when `this.phase === "finished"`
   - Reset all player state (cellId, piles, lapCount, coins, ready, etc.)
   - Reset room state (phase, ranking, currentRound, turnIndex, activePlayerId, rivers)
   - Reset cellOccupants, re-add all players to cell 1
   - Broadcast all state

### Tests

- Start over resets phase to "lobby"
- Start over resets all player state
- Start over clears ranking
- Start over ignored when phase is not "finished"

2. Remove overlay on phase "lobby" (after start over)
