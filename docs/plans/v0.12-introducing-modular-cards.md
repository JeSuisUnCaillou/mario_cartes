# v0.12 - Introducing Modular Cards

## Context

Cards are currently static types (`coin_1`, `coin_2`, `move_forward_1`, `banana_move_forward_1`) mapped to pre-made SVG assets. This rework replaces the `type` field with an `items` array so cards are composed of 1-3 items (`"coin"`, `"banana"`, `"mushroom"`). The client renders cards dynamically by overlaying item icons onto a blank card template.

## Key Design Decisions

- **Card structure**: `{ id, items: string[] }` replaces `{ id, type }`. Each item triggers one effect.
- **Dynamic rendering**: blank card background + overlaid item icons (coin.svg, banana.svg, mushroom.svg - all exist already)
- **Deck**: 8 cards (down from 9). Still draw 5, so 3 remain in draw pile.
- **`discardTopType`** becomes **`discardTopCard`** (full card object `{ id, items }`) for the discard pile rendering.
- **Pre-instantiate all cards**: The client creates DOM elements for all cards in the player's deck when it receives them. These elements are reused for all animations (draw flyer shows the real card front, not a blank placeholder).
- **Deck in player state**: `_cardState()` includes a `deck` field with all card objects (hand + drawPile + discardPile). No separate message needed — the client gets the full deck on every card state update and pre-builds DOM elements for every card, stored in a `Map<cardId, element>`.

### Slice 2: Server - new card structure, deck, and play logic

**`server/rooms/GameRoom.js`:**

1. `_createDeck()` → new 8-card deck with items arrays:
   - 3x `["coin"]`, 1x `["coin", "mushroom"]`, 1x `["banana", "coin", "mushroom"]`, 1x `["coin", "coin"]`, 1x `["banana"]`, 1x `["mushroom"]`

2. `_cardState()` → `discardTopType` becomes `discardTopCard: dp.length > 0 ? dp[dp.length - 1] : null` (full card object)

3. `playCard` handler → replace type-based if/else with items iteration:
   - Iterate `card.items`: each `"banana"` drops a banana, each `"coin"` adds 1 coin, count mushrooms for movement
   - After loop, move forward `moveCount` cells **one cell at a time**, checking for banana collision on **each cell**:
     - For each cell moved into: if banana present, remove it, add a `bananaHit` event to an array
     - After all movement, set `pendingBananaDiscards` = number of bananas hit (capped by hand size)
     - Send all banana hit events to the client so it can animate them
   - `pendingBananaDiscards` becomes a stacking counter (was effectively 0/1, now can be 0-N)
   - The `bananaHit` message changes: send an array of `bananaHits` (each with cellId) instead of a single event. Or send multiple `bananaHit` messages, one per collision.
   - Client receives banana hits and queues them. Movement animation continues regardless. Discard events stack in a queue for the player to resolve one by one.

4. `_cardState()` → add `deck: [...player.hand, ...player.drawPile, ...player.discardPile]` field containing all card objects. This lets the client pre-instantiate all card DOM elements from any card state update.

**`server/tests/lobby.test.js`:**

- Find cards by items instead of type: `c.items.length === 1 && c.items[0] === "coin"` etc.
- Add test for mushroom card moving forward

### Slice 3: Client - card pre-instantiation and modular rendering

**`client/src/player.js`:**

1. Remove `CARD_ASSETS`. Add `ITEM_ICONS = { coin: "/coin.svg", banana: "/banana.svg", mushroom: "/mushroom.svg" }`.

2. New module-level `cardElements = new Map()` to store pre-built card DOM elements by card ID.

3. New `createCardElement(card)` function:
   - Creates a `div.card` container with `data-card-id` and `data-items`
   - Appends `img.card-bg` with `card - blank.svg`
   - Appends positioned `img.card-item` icons using `cardItemPositions()`
   - Returns the element

4. New `ensureCardElements(deck)` function:
   - Iterates all cards in the `deck` array from card state
   - For any card not yet in `cardElements`, calls `createCardElement(card)` and stores it
   - Called on every `cardsDrawn` / `cardPlayed` / `cardDiscarded` message (deck is always present in card state)
   - Handles reconnect naturally — the reconnect `cardsDrawn` also includes `deck`

5. `renderHand(hand)`: instead of creating new elements, retrieves pre-built elements from `cardElements` map by card ID. Appends them to `#hand-area` with fan transforms.

6. `animateDrawCards(cards, startIndex)`:
   - Retrieves pre-built card element from `cardElements` instead of creating a new `<img>`
   - Flyer front face: clone the real pre-built card element (`cardElements.get(card.id).cloneNode(true)`) so the flip animation shows the actual card face, not a blank
   - When flyer animation ends, reveal the real card element already positioned in hand

7. `updatePiles()`: replace `discardTopType` with `discardTopCard`. When not null, build the discard icon using `createCardElement(discardTopCard)`. Apply pile sizing via CSS class.

8. Import `cardItemPositions` from `player.functions.js`.

**`client/src/style.css`:**

1. `.card`: add `position: relative; overflow: hidden;` (keep existing sizing/transitions)
2. New `.card-bg`: `width: 100%; height: 100%; border-radius: inherit; display: block;`
3. New `.card-item`: `position: absolute; width: 35%; transform: translate(-50%, -50%); pointer-events: none;`
