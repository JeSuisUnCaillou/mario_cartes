# Plan: v0.16 - Debug Mode

## Context

Need a debug modal on the board screen to inspect and edit the full game state for hand testing. The `_testSetState` message used by tests should be factored/expanded so both tests and the debug UI share the same server-side state mutation logic.

## Slice 1: Server - Expand `_testSetState` and add `_debugGetState`

**File: `server/rooms/GameRoom.js`**

1. **Broaden `_testSetState` gate and scope**: Currently gated on `this._testDeck` and only accepts `cellId`/`lapCount` from player clients. Change to:
   - Accept from player clients when `this._testDeck` is set (existing behavior)
   - Accept from board clients always (board is trusted)
   - Add `data.playerId` field so board can target any player (player clients continue to implicitly target themselves)
   - Add new editable fields: `coins`, `pendingBananaDiscards`

2. **Add `_debugSetGameState` handler** (board-only): Accepts game-level changes:
   - `phase`, `activePlayerId`, `addBanana: { cellId }`, `removeBanana: { cellId }`
   - `setRiverSlot: { riverId, slotIndex, items }` - replace a river slot card with given items (e.g. `["coin", "banana"]`)
   - Broadcasts all state after applying

3. **Add `_debugGetState` handler** (board-only): Returns full snapshot via `client.send("_debugState", snapshot)`:
   - Game: `phase`, `currentRound`, `activePlayerId`
   - `cellOccupants`
   - Players: full objects including `hand`, `drawPile`, `discardPile`, `coins`, `lapCount`, `cellId`, `pendingBananaDiscards`, `connected`, `ready`

**File: `server/tests/lobby.test.js`** - Add tests for expanded `_testSetState` and new debug messages.

## Slice 2: Client - Debug modal UI

**File: `client/src/board.js`**

- Lift `room` to module-level variable so DOM code can call `room.send()`
- Gate on `import.meta.env.DEV`: add `Ctrl+D` keyboard shortcut to toggle debug modal
- On open, send `_debugGetState` and render modal on `_debugState` response
- Modal content:
  - **Game section**: all values in editable inputs - phase (dropdown), activePlayerId (dropdown), currentRound (number input). "Apply" button sends `_debugSetGameState`
  - **Per-player sections** (listed): all values in editable inputs - name (text), cellId (number 1-14), lapCount (number), coins (number), pendingBananaDiscards (number), handCount/drawCount/discardCount (number inputs). "Apply" button per player sends `_testSetState`
  - **Circuit section**: Shows all 14 cells. Each cell shows its ID (finish line flagged), current occupants as read-only labels (player names + banana count). Each cell has an "Add banana" button. Bananas have a "Remove" button next to them. Player positions are edited via the player section's cellId input, not here.
  - **Rivers section**: Shows all 3 rivers with their cost, deck count, and 3 slot cards. Each slot shows the card items (editable - dropdown/inputs for card items like coin/mushroom/banana). Empty slots shown. Allows editing slot contents to test buy scenarios.
  - **Raw JSON toggle**: `<pre>` block with full state
- Auto-refresh: when modal is open, re-request state on each `players`/`gameState`/`cellOccupants` broadcast
- "DEBUG" badge in corner when dev mode

**File: `client/src/style.css`** - Debug modal styles (reuse `.board-leaderboard` pattern, z-index 300, scrollable)
