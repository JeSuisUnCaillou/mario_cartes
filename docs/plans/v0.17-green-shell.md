# v0.17 - Green Shell Implementation Plan

## Context

Adding a new card symbol `"green_shell"` -- the first interactive card that requires a player decision (direction) after being played. When a card with a green shell item is played, the player sees a modal to choose "Forward" or "Backward". The shell is then placed on the adjacent cell and interacts with whatever is there.

## Key Design Decisions

1. **Async card resolution**: The `playCard` handler processes all other items immediately, flags `pendingShellChoice = true`, and waits for the player to send a `shellChoice` message with the direction. This blocks `endTurn`, `playCard`, `buyCard`, `drawCards` until resolved.

2. **State over messages**: `pendingShellChoice` is stored on the player object and included in `_cardState()`. On page reload, the client sees it and re-shows the modal.

3. **Previous cell**: Compute a `this.prevCell` reverse map at `onCreate` from existing cells data (`prevCell[cell.next_cell] = cell.id`). No JSON change needed.

4. **Rename and reuse discard penalty**: Rename shared banana-named things that are now used by both banana and shell:
   - `pendingBananaDiscards` → `pendingDiscard` (server + client + tests)
   - `bananaHit` message → `discardHit` with `source` field (`"banana"` | `"green_shell"`)
   - `bananaHitBoard` message → `itemHitBoard` with `type` field (`"banana"` | `"green_shell"`)
   - `.banana-hit` CSS class → `.discard-hit` (with source-specific color via additional class `.discard-hit-banana` / `.discard-hit-shell`)

5. **Shell as cell occupant**: `"green_shell"` string in `cellOccupants` arrays, like `"banana"`. Stays on cell if nothing to hit. Players moving onto a cell with a green_shell trigger a discard, same as banana.

6. **Hit priority**: players (randomly chosen if multiple, excluding the thrower) > bananas > shells.

## New Messages

| Message | Direction | Payload |
|---------|-----------|---------|
| `shellChoice` | client->server | `{ direction: "forward" \| "backward" }` |
| `shellThrown` | server->all | `{ playerId, fromCellId, toCellId, hit: null\|"player"\|"banana"\|"green_shell", hitPlayerId? }` |
| `discardHit` | server->client | `{ source: "banana" \| "green_shell", mustDiscard, ...cardState }` |

## Slices

### Slice 0: Rename banana-specific names to generic discard/item names

- `pendingBananaDiscards` → `pendingDiscard` everywhere (server, client, tests)
- `bananaHit` message → `discardHit` (server send + client listener)
- `bananaHitBoard` message → `itemHitBoard` with `type: "banana"` field (server broadcast + board listener)
- `.banana-hit` CSS class → `.discard-hit` (player.js + style.css)
- `animateBananaHit` method → `animateItemHit` (board.js), pass item type as parameter

### Slice 1: Server -- `prevCell` map and green_shell item processing

- In `onCreate()`, build `this.prevCell` from cells data: `prevCell[cell.next_cell] = cell.id`
- Add `pendingShellChoice: false` to `_initialPlayerState()`
- Add `pendingShellChoice` to `_cardState()`, `broadcastPlayers()`, `_fullState()`
- Add guards: `if (player.pendingShellChoice) return;` in `playCard`, `drawCards`, `endTurn`, `buyCard` handlers
- In `playCard` item loop: `if (item === "green_shell") { player.pendingShellChoice = true; }`
- Add helpers: `_greenShellsOnCell(cellId)`, `_previousCell(cellId)`

### Slice 2: Server -- `shellChoice` handler, `_resolveShell`, and movement collision

- Add `shellChoice` message handler and `_resolveShell(player, targetCellId)`
- Hit priority: players (random if multiple, excluding thrower) > bananas > shells
- Movement collision: green_shell on cells triggers discard same as banana
- Add `source` field to `discardHit` messages

### Slice 3: Client player -- shell direction modal

- `showShellModal()`: "Throw the green shell" title, "Backward" (left) and "Forward" (right) buttons
- Handle in `cardPlayed` and `cardsDrawn` reconnect
- Block end turn and buy during `pendingShellChoice`

### Slice 4: Client player -- green shell hit adaptation

- Add `green_shell` to `ITEM_ICONS`
- Adapt `discardHit` handler: green crash image, `.discard-hit-shell` class, "Green shell!" text
- CSS: `.discard-hit-banana` (yellow), `.discard-hit-shell` (green `#2d8c2d`)

### Slice 5: Client board -- green shell sprites and throw animation

- Shell sprites tracked like banana sprites
- `animateShellThrow`: spawn on player, tween to target cell
- On hit banana/shell: launch both upward at 15° angle apart
- `animateItemHit` handles both banana and green_shell types

### Slice 6: Green shell in river decks and debug mode

- Add green_shell cards to river decks
- Debug: add/remove shell per cell buttons in debug modal
