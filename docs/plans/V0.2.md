# V0.2 — Setup the Game Board

## Context

Replace the current DOM-based board view with a Phaser.js scene that renders the racetrack SVG and places player helmets on cells. The "ping" button advances that player one cell forward. Game data introduces a 14-cell chained list (cell 1 = checkered/finish line) with each player having a `cellId`.

## Slices

### Slice 0: Phaser + static racetrack display

1. Configure `client/vite.config.js`: set `publicDir: '../assets'` so Vite serves `/racetrack.svg` and `/helmet.svg`
2. Install Phaser: `npm install phaser -w client`
3. Replace `client/src/board.js` with a Phaser 3 scene:
   - Canvas fills the window, `Scale.FIT` + `CENTER_BOTH`
   - Load `racetrack.svg` with `{ scale: 4 }`, display centered

### Slice 1: Game data + player helmets + movement

**Server (`GameRoom.js`)**:
- Add `cellId: 1` to player entries in `clientsInfo`
- On `"ping"`: advance player's cellId (`cellId % 14 + 1`), broadcast
- Include `cellId` in `broadcastPlayers()` payload

**Client (`board.js`)**:
- Define `CELL_POSITIONS` (14 cells as [fractionX, fractionY] of SVG viewBox):
  ```
  1:(0.096,0.374) 2:(0.096,0.120) 3:(0.298,0.120) 4:(0.500,0.120)
  5:(0.702,0.120) 6:(0.904,0.120) 7:(0.904,0.374) 8:(0.904,0.626)
  9:(0.904,0.880) 10:(0.702,0.880) 11:(0.500,0.880) 12:(0.298,0.880)
  13:(0.096,0.880) 14:(0.096,0.626)
  ```
- On `"players"` message: create/move/destroy helmet images per player
- Cell pixel position: `trackImage.x - trackImage.displayWidth/2 + fx * trackImage.displayWidth`

## Verification

1. `npm run dev` from project root
2. Create game → `/game/:id/board` shows Phaser canvas with racetrack
3. Open `/game/:id/player` → click ping → helmet appears and moves through cells
4. 14 pings → helmet wraps back to cell 1
5. Multiple player tabs → each has its own helmet
