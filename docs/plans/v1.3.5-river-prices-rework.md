# v1.3.5 — River prices rework

## Context

Currently, river access is rank-gated: lower-ranked players are forbidden from buying certain rivers. This creates frustrating "red bar" experiences. The rework replaces binary access control with a rank-scaled pricing formula, so every river is always accessible but costs more for leading players.

**New formula:** `effective_price = base_price + (nb_players - rank) × price_scale_factor`
- `price_scale_factor = 1`
- Rank starts at 1 (no rank 0)
- Rank 1 (leader) pays the most: `base + (nb_players - 1)`
- Last place (rank = nb_players) pays just the base price

## Changes

### 1. New shared pricing function
**Files:** `server/rooms/riverRules.js`, `client/src/river.functions.js` (kept in sync)

- Replace `canBuyFromRiver` with `getRiverPrice(baseCost, rank, playerCount)`:
  ```js
  const PRICE_SCALE_FACTOR = 1;
  function getRiverPrice(baseCost, rank, playerCount) {
    return baseCost + (playerCount - rank) * PRICE_SCALE_FACTOR;
  }
  ```
- Ranks start at 1 (leader) through nb_players (last place)
