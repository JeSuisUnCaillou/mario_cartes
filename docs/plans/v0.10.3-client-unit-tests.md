# Plan: Unit Tests for Board & Player UI — Game State Functions

## Context

The project has zero client-side tests. `board.js` and `player.js` contain some pure functions that transform or represent game state, mixed in with DOM/Phaser code. Extracting these enables unit testing. We only test game state representation — not visual layout or positioning.

## What to extract

### Board — `client/src/board.functions.js`

| Function | Source | Logic |
|---|---|---|
| `bananaCounts(cellOccupants)` → `{[cellId]: count}` | board.js:334-338 | Filters occupant arrays to count bananas per cell |

### Player — `client/src/player.functions.js`

| Function | Source | Logic |
|---|---|---|
| `isPointInRect(px, py, left, top, right, bottom)` → `boolean` | player.js:285-289 | AABB hit test — determines if card drop is valid |
| `splitDrawBatches(hand, drawnBeforeShuffle)` → `{firstBatch, secondBatch}` | player.js:576-578, 689-691 | Splits hand for draw sequencing (duplicated in 2 handlers) |
| `initialDrawPileCount(drawCount, handLength, shuffledCount, drawnBeforeShuffle)` → `number` | player.js:581-584, 692 | `shuffledCount > 0 ? drawnBeforeShuffle : drawCount + handLength` (duplicated) |
| `normalizeName(raw)` → `string` | player.js:457, 523 | `toUpperCase().replace(/[^A-Z]/g, "")` (duplicated) |

### Not extracting

- Fan transforms, cell slot positions, pixel positions, layout scales, center-in-rect — all visual-only
- `getPointerPos`, `renderPileContent`, `updatePileCount` — DOM-only, no game state
