# v0.7 - Introduce Cards: Implementation Plan

## Context

The game needs its core mechanic: cards. Currently players move by clicking a "Ping" button. This replaces that with a deck-based system where players draw cards and drag them to a play zone. Each player starts with 8 "Move forward 1 cell" cards in their draw pile, draws 5 at a time, and plays them by dragging to a drop zone.

## Design Decision: Vanilla JS (not Phaser) for player screen

The player screen stays vanilla HTML/JS + CSS. The card fan uses CSS transforms, and drag-and-drop uses touch events (`touchstart`/`touchmove`/`touchend`) + mouse fallback. Rationale: this is a mobile UI, not a game loop; Phaser would double the bundle for no benefit.

## Message Protocol

| Message | Direction | Payload | Effect |
|---------|-----------|---------|--------|
| `"drawCards"` | Client → Server | (none) | Move top 5 from drawPile → hand |
| `"cardsDrawn"` | Server → Client | `{ hand, drawCount, discardCount }` | Client renders cards |
| `"playCard"` | Client → Server | `{ cardId }` | Resolve action, card → discard |
| `"cardPlayed"` | Server → Client | `{ cardId, hand, drawCount, discardCount }` | Client updates UI |

## Slices

### Slice 1 — Backend: Card state + draw mechanic

**File: `server/rooms/GameRoom.js`**

- Add `_createDeck()` helper: returns array of 8 cards `{ id: randomUUID(), type: "move_forward_1" }`
- In `onJoin` (new player path): add `drawPile: _createDeck()`, `hand: []`, `discardPile: []` to player object
- Add `"drawCards"` message handler:
  - Guard: `player.hand.length > 0` → ignore
  - `splice(0, 5)` from `drawPile` into `hand`
  - Send `"cardsDrawn"` to client: `{ hand, drawCount, discardCount }`
- On reconnect: if player has cards in hand, send `"cardsDrawn"` with current hand
- `broadcastPlayers()` must NOT include card data (cards are private)
- Remove `"ping"` message handler

### Slice 2 — Backend: Play card mechanic

**File: `server/rooms/GameRoom.js`**

- Add `"playCard"` message handler:
  - Find card by `cardId` in `player.hand`. If not found, ignore.
  - Remove card from hand
  - Resolve action: `"move_forward_1"` → advance `cellId` to `next_cell`
  - Push card to `discardPile`
  - Send `"cardPlayed"` to client: `{ cardId, hand, drawCount, discardCount }`
  - Call `broadcastPlayers()` (so board sees position update)

### Slice 3 — Frontend: 3-zone layout (no interaction yet)

**File: `client/src/player.js`**

Replace `startGame()` HTML with:
- **Top**: Name input (keep existing)
- **Middle**: Play zone (dashed border, "Drag a card here")
- **Bottom** (flexbox row):
  - Left: Draw pile (card back icon + count + "Draw" button when hand empty)
  - Center: Hand area (empty)
  - Right: Discard pile (card face icon + count)

Remove Ping button and cell-info.

**File: `client/src/style.css`**

- `.player-screen`: flex column, `height: 100dvh`
- `.play-zone`: flex-grow middle area, dashed border
- `.cards-zone`: fixed bottom area (~40vh)
- `.draw-pile`, `.hand-area`, `.discard-pile`: sub-zones

### Slice 4 — Frontend: Draw cards + fan layout

**File: `client/src/player.js`**

- Wire "Draw" button → `room.send("drawCards")`
- Listen for `"cardsDrawn"`: call `renderHand(hand)`
- `renderHand(hand)`:
  - Create `<img src="/card - move forward.svg">` per card
  - Fan layout via CSS transforms:
    - `transform-origin: bottom center`
    - Each card rotated: `(i - (N-1)/2) * 10deg`
    - Slight Y offset for arc: `abs(i - (N-1)/2) * 5px`
  - Negative margins for overlap
- Show/hide Draw button based on hand length
- Update pile counts

### Slice 5 — Frontend: Drag to play zone

**File: `client/src/player.js`**

- Touch + mouse event handlers on each card:
  - `touchstart/mousedown`: lift card (`position: fixed`, scale up, z-index)
  - `touchmove/mousemove`: follow pointer
  - `touchend/mouseup`: hit-test against play zone rect
    - **Hit**: snap to zone center (CSS transition), wait ~1s, send `"playCard"`
    - **Miss**: animate back to fan position
- Listen for `"cardPlayed"`: re-render hand, update piles, show Draw button if hand empty
- Add `touch-action: none` on cards to prevent scroll during drag
- Add a `playing` flag to prevent concurrent plays
