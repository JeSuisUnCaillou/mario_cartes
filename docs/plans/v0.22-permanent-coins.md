# v0.22 — Permanent Coins

## Context

Cells 3, 7, and 12 have a `permanent_coin: 1` flag in `racetrack_0_cells.json`. A `permacoin.svg` asset (blue coin) already exists. The feature adds a new coin type that players collect by landing on these cells. Unlike regular coins (reset to 0 at turn end), permanent coins are restored after spending.

**Key design decision — coin model:**
- `player.permanentCoins` = total permanent coins earned (grows over the game, never decreases)
- `player.coins` = total spendable coins (regular + permanent combined)
- Spending deducts from `coins` only → regular coins are spent first naturally
- At turn end: `coins = permanentCoins` (instead of `coins = 0`)
- UI shows: first `min(coins, permanentCoins)` blue coins, then `max(0, coins - permanentCoins)` gold coins

**Board rendering:** Permanent coins are static (never consumed from the cell). They are NOT in `cellOccupants`. The client prepends a `"permacoin"` entry to the local occupants array for cells that have one, so existing grid layout code handles positioning automatically.

  player.permanentCoins += cellData.permanent_coin;
