# v0.13 — Introduce Card Buying Mechanic (Rivers)

## Context

Players earn coins by playing cards but have no way to spend them. This version adds "rivers" — shared card pools displayed on the board — where players can spend coins during their turn to buy new cards and add them to their discard pile.

## River Card Definitions

Each river has a fixed deck. On game start, 3 cards are drawn to visible slots; the rest stay in a hidden deck.

**River 1 (cost: 1 coin) — 6 cards:**
- `["coin"]`, `["coin"]`, `["mushroom"]`, `["mushroom"]`, `["banana"]`, `["banana"]`

**River 2 (cost: 3 coins) — 6 cards:**
- `["coin", "mushroom"]`, `["coin", "banana"]`, `["mushroom", "banana"]`, `["mushroom", "mushroom"]`, `["banana", "banana"]`, `["banana", "banana"]`

**River 3 (cost: 5 coins) — 4 cards:**
- `["banana", "banana", "mushroom"]`, `["banana", "banana", "coin"]`, `["coin", "coin", "mushroom"]`, `["coin", "coin", "banana"]`

## Slices

### Slice 1: Server — River state & buyCard handler + tests

**Files:** `server/rooms/GameRoom.js`, `server/tests/lobby.test.js`, `server/tests/helpers.js`

1. Add `_createRiverDecks()` method returning 3 river objects:
   ```js
   [
     { id: 0, cost: 1, deck: [...], slots: [card, card, card] },
     { id: 1, cost: 3, deck: [...], slots: [card, card, card] },
     { id: 2, cost: 5, deck: [...], slots: [card, card, card] },  // only 1 in deck
   ]
   ```
   - Each card gets a `randomUUID()` id, like player cards
   - Decks are shuffled, then 3 cards drawn into `slots`
   - Accept `_testRiverDecks` option for deterministic tests

2. Call `_createRiverDecks()` in `_startGame()`, store as `this.rivers`

3. Add `_riverState()` — returns public view (slots + deckCount, no hidden deck):
   ```js
   this.rivers.map(r => ({ id: r.id, cost: r.cost, slots: r.slots, deckCount: r.deck.length }))
   ```

4. Add `broadcastRivers()` — broadcasts `"rivers"` message to all clients

5. Broadcast rivers from: `_startGame()` and `onJoin()` (when phase is "playing")

6. Add `"buyCard"` message handler:
   - Validate: phase === "playing", player is active, no pending banana discards
   - Find river by `data.riverId`, find card in `river.slots` by `data.cardId`
   - Validate: card exists, `player.coins >= river.cost`
   - Deduct coins, move card from slot to `player.discardPile`
   - Refill slot from `river.deck` (or set to `null` if empty)
   - Send `"cardBought"` to player: `{ cardId, riverId, slotIndex, ...this._cardState(player) }`
   - Call `broadcastRivers()` and `broadcastPlayers()`

7. In `server/tests/helpers.js`: add `"cardBought"` and `"rivers"` to `BUFFERED_TYPES`

8. Tests in `server/tests/lobby.test.js` — new `describe("Card buying (rivers)")`:
   - Rivers broadcast on game start with correct structure
   - Rivers broadcast on reconnect
   - buyCard succeeds with enough coins (card ends up in discardPile, coins deducted)
   - buyCard fails with insufficient coins
   - buyCard fails when not player's turn
   - buyCard refills slot from deck
   - buyCard leaves slot null when deck is empty

### Slice 3: Player client — Buy button & modal

**Files:** `client/src/player.js`, `client/src/style.css`

1. Add module-level `latestRivers = null` to store latest river state.

2. Add `"rivers"` message listener in `setupRoom()` — stores in `latestRivers`, updates buy button visibility.

3. Add `"cardBought"` message listener:
   - Update piles via existing `updatePiles()` pattern
   - Update coin display via `updateCoinDisplay(data.coins)`
   - Call `ensureCardElements(data.deck)` for the newly acquired card
   - Animate bought card to discard pile (clone from modal position → discard pile)
   - Update `latestRivers` locally (remove bought card from slot, add refilled card if server sends updated rivers)
   - Re-render modal if open

4. Add "Buy cards" button below the coin display (`#coin-display`):
   - Placed after the coin display element, not in the end-turn-container
   - Visible only when it's the player's turn
   - Available anytime during turn (before, between, or after playing cards)

5. Add `openBuyModal()`:

   - Null slots show empty placeholder
   - Tapping affordable card sends `room.send("buyCard", { riverId, cardId })`
   - Modal stays open after purchase (player may buy more)

6. Discard animation for bought cards:
   - Clone the tapped card element at its screen position
   - Animate clone to discard pile position (reuse existing discard animation pattern from card play flow)

7. CSS: `.buy-modal`, `.buy-modal-close`, `.buy-modal-river`, `.buy-modal-card`, `.buy-modal-card.unaffordable`, `.buy-btn`

## Key Constraints

- Server is authority: all validation happens server-side
- UI handles reload: rivers broadcast on reconnect, modal can be reopened
- Cards have unique UUIDs
- Coins already persist during turn (reset only in `_endTurnAndAdvance`)
- Buy available anytime during turn
