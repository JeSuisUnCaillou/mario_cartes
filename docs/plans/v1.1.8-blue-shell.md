# v1.1.8 — Blue Shell

## Context

Add the `blue_shell` card. When played, it automatically travels forward to the rank-1 player (no direction choice) and forces them to discard their entire hand. It skips all obstacles and players along the way — it only hits rank 1. The asset (`blue_shell.svg`) and deck entry (river_4, `2x blue_shell`) already exist.

## Slices

### 1. Plan — save this plan to `docs/plans/`

### 2. Server: `_resolveBlueShell` + route in `_processNextItem`

**`server/rooms/GameRoom.js`**

- Add `_resolveBlueShell(thrower)` after `_resolveRedShell` (line 268):
  1. Compute live ranks (same pattern as `_getLiveRank`, line 735-742)
  2. Find player with rank 1 among **unfinished** players (exclude `this.ranking` entries)
  3. Build forward path from thrower cell to target cell via `next_cell` (skip all occupants — no hit checks)
  4. **Auto-discard**: move all cards from `target.hand` to `target.discardPile` server-side. No `pendingDiscard` — the discard is instant and automatic.
  5. Send `blueShellHit` message to target with `{ discardedCardIds, ...cardState }` so client can animate the cards leaving
  6. Broadcast `shellThrown` with `shellType: "blue_shell"`, `path`, `hit: "player"`, `hitPlayerId`

- Route in `_processNextItem` (line 921): add `else if (item === "blue_shell")` before the green/red shell block. Call `_resolveBlueShell(player)`. No need to check `pendingDiscard` — the discard is server-side and instant, so always fall through to the `setTimeout` at line 929.
