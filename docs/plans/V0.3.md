# V0.3 — Improve Player Movement

## Goal

Associate the racetrack image with structured cell data (id, center_position, next_cell) stored in a JSON file. The backend loads this data and uses it to determine movement — instead of the hardcoded `% 14 + 1` formula.

### Slice 3 — Backend loads cells and uses next_cell for movement

In `server/rooms/GameRoom.js`:
- On `onCreate`, load `racetrack_0_cells.json` from the assets folder and build a `Map<cellId, cellData>` (call it `this.cells`)
- In the `"ping"` handler, replace `info.cellId = (info.cellId % 14) + 1` with:
  ```javascript
  info.cellId = this.cells.get(info.cellId).next_cell;
  ```

The assets folder is at `../../assets/racetrack_0_cells.json` relative to `server/rooms/GameRoom.js`. Use `require` (or `fs.readFileSync` + `JSON.parse`) to load it at room creation time.
