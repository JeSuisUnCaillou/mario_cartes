# v0.24 — Upgrade Colyseus 0.15 → 0.17

## Why

Colyseus 0.17 brings built-in auto-reconnection (critical for phone-lock), cleaner lifecycle hooks (`onDrop`/`onReconnect`), and modern `schema()` syntax.

## Changes

### Slice 1: Dependencies + ESM + imports
- `colyseus` ^0.17.8, `@colyseus/sdk` ^0.17.36 (replaces `colyseus.js`), `@colyseus/schema` ^4.0
- Server converted from CommonJS to ESM
- Server uses `Server({ express })` pattern (replaces manual httpServer setup)

### Slice 2: Schema definitions
- `defineTypes()` → `schema()` function in `server/rooms/schema.js`

### Slice 3: Server lifecycle
- `onLeave(client, consented)` → split into `onDrop`/`onReconnect`/`onLeave`
- `allowReconnection(client, 120)` moves to `onDrop`

### Slice 4: Client state callbacks
- `room.state.listen()` / `room.state.players.onAdd()` → `Callbacks.get(room)` API

### Slice 5: Client reconnection
- Remove manual reconnection code, use built-in `room.reconnection` with `maxRetries: 30`
- Keep fallback fresh-join on code 4003 (reconnection exhausted)
