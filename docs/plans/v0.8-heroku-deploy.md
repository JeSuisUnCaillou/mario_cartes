# V0.8 — Heroku Deployment

## Goal

Make the app deployable on Heroku with a single `git push`. The server must build the client, serve its static files, and handle SPA routing.

## Current State

- Monorepo with `server/` and `client/` workspaces
- Dev mode: Vite dev server proxies API calls to Express+Colyseus on port 2567
- No `Procfile`, no build script, server doesn't serve static files
- WebSocket URL logic in client already handles production (`wss://`)
- API routes (`/create`, `/find-or-create`) are on the Express server

## What Needs to Change

### Slice 1 — Build pipeline & static serving

1. **Root `package.json`**: add a `"build"` script that builds the client:
   ```
   "build": "npm run build -w client"
   ```
   Heroku automatically runs `npm run build` after `npm install`.

2. **Root `package.json`**: add `"engines"` field to pin Node version:
   ```
   "engines": { "node": "24.x" }
   ```

3. **Server `index.js`**: serve `client/dist` as static files and add a catch-all route for SPA client-side routing (serves `index.html` for any non-API, non-file request).

4. **`Procfile`**: tell Heroku how to start:
   ```
   web: npm start -w server
   ```

### Slice 2 — Verify locally

Run a production-like build+start locally and confirm:
- `npm run build` produces `client/dist/`
- `npm start -w server` serves the app on `$PORT`
- `/`, `/game/:id/board`, `/game/:id/player` all load the SPA
- `/create` and `/find-or-create/:id` still return JSON
- WebSocket connection works

## Out of Scope

- Custom domain, SSL (Heroku handles this)
- Database or persistence
- CI/CD pipeline
