# Plan: v0.5 — Player Name

## Goal

Each player enters a 3-character uppercase name (Super Smash Bros. style) on their screen. The name is stored in the backend and displayed above the player's helmet on the board.

## Slices

### Slice 1 — Name input on player screen + backend storage

**Player screen (`client/src/player.js`):**
- Before joining the room, show a form with:
  - A single text input: max 3 chars, auto-uppercased, no spaces
  - A submit button
- On submit, join the room passing `{ type: "player", name }` as options

**Backend (`server/rooms/GameRoom.js`):**
- In `onJoin`, read `options.name` (sanitize: uppercase, trim, max 3 chars, default `"???"`), store in player info
- Include `name` in `broadcastPlayers()` output

### Slice 2 — Name label above helmet on board

**Board (`client/src/board.js`):**
- Add a `nameLabels` Map alongside `helmets`
- When creating a helmet, also create a Phaser `Text` object above it with the player's name
- On position update, move the label with the helmet
- On player leave, destroy the label with the helmet
