# V0.11 - Introducing Coins

## Context

The game currently has 2 card types (`move_forward_1`, `banana_move_forward_1`) in a 10-card deck. Turns end automatically when all cards are played. This version introduces coin cards, a per-turn coin economy, an explicit "End turn" button, and coin displays on both board and player screens.

## Key Design Decisions

- **New card types**: `coin_1` and `coin_2` (no movement effect, just add coins)
- **New deck**: 9 cards (6x `coin_1`, 1x `coin_2`, 1x `move_forward_1`, 1x `banana_move_forward_1`). Still draw 5.
- **Coins are per-turn**: reset to 0 when the turn ends
- **End turn is always explicit**: player can end turn anytime (even with cards remaining). Turns never auto-end, not even after banana discard resolution.
- **Server is authority**: server tracks `coins`, client renders what server sends
- **Throw animation factorized**: banana and coin share a `spawnThrowAnimation()` function. Animation reworked: no fade/shrink, goes off-screen top, rotates 720deg.

## Critical Files

- `server/rooms/GameRoom.js` -- all server game logic
- `server/tests/lobby.test.js` + `server/tests/helpers.js` -- server tests
- `client/src/player.js` -- player screen
- `client/src/board.js` -- board screen
- `client/src/style.css` -- all styling
