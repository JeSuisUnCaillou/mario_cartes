# v0.20 ‚Äî Live Ranking

## Context

Players currently have no visibility into their position relative to others during the race. Rank is only shown when a player finishes. This feature adds a **live rank** computed after every move, so players can see their standing at all times during gameplay.

The existing finish ranking (`this.ranking` array / `RankEntrySchema`) is renamed to `finalRank` to distinguish it from the new live rank.

## Slice 2 ‚Äî Add `rank` field to PlayerSchema + compute live ranks

### Server ‚Äî schema
- `server/rooms/schema.js:4-14` ‚Äî add `rank: "number"` to `PlayerSchema`

### Server ‚Äî ranking logic
- `server/rooms/GameRoom.js` ‚Äî add `_computeLiveRanks()` method:
  ```
  _computeLiveRanks():
    1. Start with finished players: assign their final rank as live rank
       (e.g. if ranking = [P1, P2], then P1.rank=1, P2.rank=2)
    2. For each unfinished player, compute position score:
       - If lapCount === 0: score = 0
       - Else: score = (lapCount - 1) * this.cells.size + (cellId - 1)
    3. Sort unfinished players by score descending
    4. Assign ranks with ties (standard competition ranking):
       - The first unfinished player's rank starts at finishedCount + 1
       - Same score = same rank, next rank skips
    5. Return Map<playerId, rank>
  ```
  Example: 4 players, P1 finished (finalRank=1), P2 on cell 5 lap 2, P3 on cell 5 lap 2, P4 on cell 1 lap 1:
  ‚Üí P1: rank 1 (from final rank), P2: rank 2, P3: rank 2 (tied), P4: rank 4
- `server/rooms/GameRoom.js` ‚Äî in `_syncSchemaFields()` (line ~634-652):
  - Call `_computeLiveRanks()` 
  - Set `sp.rank = liveRanks.get(playerId)` for each player
  - Only compute during `playing` phase (rank = 0 in lobby/finished)

### Tests
- `server/tests/helpers.js:14-31` ‚Äî add `rank` to `schemaPlayersToArray` output
- New test file `server/tests/live-ranking.test.js`:

### Extract pure function

  - `players`: array of `{playerId, cellId, lapCount}`
  - `finishedRanks`: array of `{playerId, finalRank}` (from the finish ranking, in order)
  - `cellCount`: number of cells on the track

- This keeps it testable without needing a full game room
