# V0.4 - Harden Room Reload

## Goal

Make the game resilient to reloads and lost connections:
- Board at `/game/:uid/board` creates a room if none exists for that UID
- Player at `/game/:uid/player` retries joining until the room appears
- Rooms survive up to 10 minutes after the last client leaves (no instant teardown)

## Slices

### Slice 1 — Room grace period

**server/rooms/GameRoom.js**:
- Set `this.autoDispose = false` in `onCreate` to prevent instant teardown
- In `onLeave`, if no clients remain, schedule `this.disconnect()` after 10 minutes
- In `onJoin`, cancel any pending destroy timer

### Slice 2 — Board creates room if missing

**server/index.js**:
- Add `GET /find-or-create/:gameId` endpoint:
  - Query rooms by `roomId` via `matchMaker.query({ roomId: gameId })`
  - If found: return `{ id: gameId }`
  - If not found: create new room and return its id

**client/src/board.js**:
- Wrap the `joinById` call in a try/catch
- On failure: fetch `/find-or-create/:gameId`, update the URL silently with
  `history.replaceState`, then join the returned room id

### Slice 3 — Player waits for room

**client/src/player.js**:
- Add `joinWithRetry(client, gameId, options)` that loops up to 30 times
  with a 2-second delay between attempts
- Show a "Waiting for game to start..." message while retrying
- Show an error if all attempts are exhausted
