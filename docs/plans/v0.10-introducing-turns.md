# V0.10 - Introducing Turns: Implementation Plan

## Context

The game currently has no concept of turns, rounds, or game start. All players can play cards simultaneously at any time, and hands auto-redraw the moment they're empty. This plan introduces:
1. A **ready system** (lobby phase before the game starts)
2. **Turn-based play** where each player plays their entire hand before the next player's turn
3. **Round tracking** displayed on the board

## Key Design Decisions

**Game phases**: `"lobby"` (waiting for all ready) → `"playing"` (game started). No going back.

**Turn flow**:
1. Game starts → server draws 5 cards for ALL players (everyone sees the deal animation)
2. First player is active → plays all their cards one by one
3. When the active player has played all their original cards AND any banana discards are resolved → turn ends → turn advances to next player
4. Next player already has cards → plays → same cycle
5. When all players have played in a round → round number increments → first player goes again

**Redraw at end of turn**: only if the player's hand is empty. If a banana hit caused an auto-draw (giving them cards to discard from), those remaining cards **are** their next-turn hand — no additional redraw.

**Banana-on-last-card edge case**: Player plays card #5 → hand empty → lands on banana → server auto-draws 5 (so they can discard 1) → player discards 1 → has 4 cards → turn ends. Those 4 cards are their hand for next turn. No redraw.

**Turn-end detection**: Uses a `hasPlayedAllCards` flag on the player object. Set to `true` when `hand.length === 0` after a `playCard` (before any banana auto-draw). Turn ends when `hasPlayedAllCards === true && pendingBananaDiscards === 0`. Reset to `false` at turn start.

**Auto-draw is disabled during play**. The only draws are: (a) initial deal at game start, (b) redraw at end of turn (only if hand is empty), (c) banana-hit auto-draw when hand is empty and player must discard.

**Server enforces turns**: `playCard`, `drawCards`, `discardCard` are rejected for non-active players.

**Turn order**: iteration order of `this.players` Map (same as the player list displayed on the board).

## Critical Files

- `server/rooms/GameRoom.js` — all server game logic
- `client/src/player.js` — player screen UI
- `client/src/board.js` — board screen UI
- `client/src/style.css` — styling

## New/Modified Messages

| Message | Direction | Payload | Purpose |
|---------|-----------|---------|---------|
| `setReady` (new) | Client → Server | `boolean` | Toggle ready state |
| `gameState` (new) | Server → All | `{ phase, currentRound, activePlayerId }` | Broadcast game/turn state |
| `players` (modified) | Server → All | adds `ready` field | Include ready state |

## Slices

### Slice 1 — Server: game phase, ready state, and game start

**`server/rooms/GameRoom.js`**

- Add to `onCreate()`:
  - `this.phase = "lobby"`
  - `this.currentRound = 0`
  - `this.turnIndex = -1`
  - `this.activePlayerId = null`

- Add `ready: false` and `hasPlayedAllCards: false` to player object in `onJoin` (line 198)

- Add `ready: p.ready` to `broadcastPlayers()` output (line 236)

- New `setReady` message handler:
  - Guard: only in lobby phase
  - Set `player.ready = !!ready`
  - Broadcast players
  - Call `_checkAllReady()`

- New `_checkAllReady()`:
  - If `players.size > 0` and all have `ready === true` → call `_startGame()`

- New `_startGame()`:
  - Set `phase = "playing"`, `currentRound = 1`, `turnIndex = 0`
  - Set `activePlayerId` to first player key
  - Draw 5 cards for ALL players, send `cardsDrawn` to each
  - Broadcast `gameState` and `players`

- New `broadcastGameState()`:
  - Broadcasts `{ phase, currentRound, activePlayerId }`

- New `_sendToPlayer(playerId, type, data)`:
  - Find client by iterating `clientsInfo` to match playerId, then find client in `this.clients`

- Call `broadcastGameState()` in `onJoin` (so new clients get current state)

- Block new player creation if `phase === "playing"` (reconnecting existing players is still allowed)

### Slice 2 — Server: turn tracking and enforcement

**`server/rooms/GameRoom.js`**

- Guard `playCard` handler: reject if `player.playerId !== this.activePlayerId`

- Guard `drawCards` handler: reject if `player.playerId !== this.activePlayerId`

- Guard `playCard`, `drawCards`, `discardCard`: reject if `phase !== "playing"`

- In `playCard` handler, after removing card from hand: if `player.hand.length === 0`, set `player.hasPlayedAllCards = true` (BEFORE banana check, so the flag is set even if banana auto-draw refills the hand)

- After `playCard` processing (after banana check completes): if `player.hasPlayedAllCards && player.pendingBananaDiscards === 0`, call `_endTurnAndAdvance(player)`

- After `discardCard` processing: if `player.hasPlayedAllCards && player.pendingBananaDiscards === 0 && player.playerId === this.activePlayerId`, call `_endTurnAndAdvance(player)`

- New `_endTurnAndAdvance(player)`:
  1. If `player.hand.length === 0`: redraw 5 cards for the player (for their next turn), send `cardsDrawn`
  2. If `player.hand.length > 0`: skip redraw (banana auto-draw already gave them cards — those are their next-turn hand)
  3. Reset `player.hasPlayedAllCards = false`
  4. Call `_advanceTurn()`

- New `_advanceTurn()`:
  1. Increment `turnIndex`; if past end, reset to 0 and increment `currentRound`
  2. Set `activePlayerId` from player keys at new index
  3. Skip disconnected players (loop with safety counter)
  4. Broadcast `gameState` and `players`

- At turn start (in `_startGame` and `_advanceTurn`): reset `hasPlayedAllCards = false` for the new active player

- `onLeave`: if active player disconnects during "playing" phase, call `_advanceTurn()`

### Slice 3 — Client player: ready screen

**`client/src/player.js`**

- Add module-level state: `let gamePhase = "lobby"`, `let isReady = false`
- Move `myPlayerId` to module level (currently scoped inside `startGame`)

- Restructure `startGame()` HTML:
  - Keep top-zone with name input
  - Add `#lobby-zone` with READY button (shown initially)
  - Wrap play-zone + cards-zone in `#game-zone` (hidden initially)

- READY button: sends `room.send("setReady", true)`
- Cancel button: sends `room.send("setReady", false)`

- Listen for `players` message in lobby to update own ready state display (in case of reconnect)

- Add `gameState` message handler:
  - Update `gamePhase`
  - When `phase === "playing"`: hide lobby-zone, show game-zone

**`client/src/style.css`**

- `.lobby-zone`: flex column, centered, flex 1
- `.ready-btn`: large font, big padding
- `.ready-message`: large bold green text
- `.cancel-btn`: regular size

### Slice 4 — Client player: turn state in play zone

**`client/src/player.js`**

- `gameState` handler also stores `activePlayerId` and calls `updatePlayZone()`

- New `updatePlayZone()`:
  - If `activePlayerId === myPlayerId` AND not in banana-hit mode: normal play zone ("Drag a card here to play it")
  - If `activePlayerId !== myPlayerId`: add `.waiting` class, show "Wait for your turn to play"

- Modify drag `onEnd`: if play zone has `.waiting` class, treat drop as miss (snap back to hand)

- Remove `autoDrawIfEmpty()` calls from `cardPlayed` and `cardDiscarded` handlers
- Remove the `setTimeout(() => room.send("drawCards"), 2000)` auto-draw on join
- Delete `autoDrawIfEmpty` function entirely since it's unused.

**`client/src/style.css`**

- `.play-zone.waiting`: muted border color, lower opacity text

### Slice 5 — Board: round card and active player indicator

**`client/src/board.js`**

- Add `gameState` message listener in `connectToRoom()`:
  - On `phase === "playing"`: remove `.board-scan-label`, insert `.board-round-card` as first child of players row
  - Update round number text on each `gameState`
  - Toggle `.active-player` class on the matching `.board-player` element

- `.board-round-card`: 80px wide (same as other elements), displays "Round" + round number in bold, vertically centered

**`client/src/style.css`**

- `.board-round-number`: large font (2em)
- `.board-player.active-player`: 2px solid gold border, small border-radius

### Slice 6 — Edge cases and reconnection

**`server/rooms/GameRoom.js`**

- `onJoin` for reconnecting player during "playing" phase: send current `gameState` directly to that client
- `_advanceTurn`: skip disconnected players with a loop guard (prevent infinite loop if all disconnected)
- `onLeave`: only advance turn if leaving player is the active player

**`client/src/player.js`**

- On reconnect during "playing" phase: the `gameState` message will arrive and set the correct UI state (lobby hidden, game shown, play zone enabled/disabled)
- Handle the reconnect `cardsDrawn` (the one without `drawnBeforeShuffle`) — this already renders without animation, just ensure play zone state is correct afterward

## Verification

1. Start server + client. Two browser tabs join as players.
2. Both see "READY" button. Board shows "Scan to join" and both players.
3. Player A hits READY → sees "You are ready" + Cancel. Player B still sees READY button.
4. Player A hits Cancel → back to READY button.
5. Both hit READY → game starts. Board replaces "Scan to join" with "Round 1". Golden border on Player A.
6. Player A sees cards dealt (animation), can play. Player B sees cards dealt, play zone says "Wait for your turn to play".
7. Player A plays all 5 cards → redraws 5 (animation) → golden border moves to Player B.
8. Player B plays all → redraws → round becomes 2 → Player A's turn again.
9. Disconnect test: Player A disconnects mid-turn → turn skips to Player B → Player A reconnects and waits.
10. Banana mid-turn: active player hits banana with cards remaining → must discard → continues playing remaining cards → turn ends when hand empty → redraws.
11. Banana on last card: active player plays card #5, lands on banana → auto-draw 5 → discards 1 → has 4 cards → turn ends with NO additional redraw → those 4 cards are next-turn hand.
