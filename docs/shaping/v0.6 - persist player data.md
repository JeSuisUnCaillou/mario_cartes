# v0.6 - persist player's data

The player's client should display his current data (typically its cell_id).

When a player reloads his page, if the game already knew him, he should find himself in the same position as before he reloaded the page.

For that, the backend should keep a list of the players who have joined and their positions and names separated from the connected players. This should be the source of truth for player data.
