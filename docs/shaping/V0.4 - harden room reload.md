# V0.4 - harden room reload

When a client opens the board's path with an inexistant game id, the room should be created.
When a client opens the player's path with an inexistant game id, it should wait for it to be created and then join it.

This will harden the game's behavior when reloading or when internet connection is lost. 

Also, a room should not be destroyed right away if no board or players are active. you can keep the room up for like 10min before destroying it if there is no board or players.
