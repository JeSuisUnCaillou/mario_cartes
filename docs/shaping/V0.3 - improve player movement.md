# Improve player movement

The racetrack image file should be associated with an array of all its cells data.

Each cell data contains : 
- id (1 to 14 for the only existing racetrack)
- center_position : coordinates of its center, around which we will position all the player's helmets
- next_cell: the id of the next cell (the 14's next is 1)

-> rename racetrack.svg into racetrack_0.svg.
-> Create the json file of cell data in the assets folder, name ot racetrack_0_cells.json. You should be able to compute the center of each cell from the svg's data.

The backend should load the racetrack's cells and keep it in the game data.

The backend should not increment the player's cell_id on ping, but instead should look at its current cell's data to know the next cell id to go to.
