# v1.0 - Slow mechanic for shells

New mechanic : the slow counters. They are symbolised with a dark mushroom asset.

All player starts with 0 slow counters. The game reset also reset them to 0.

when a player is hit by a shell, instead of discarding, now he gains a slow counter.

In the board, the slow counters are displayed next to the coins of each players (not displayed if they have none). With a number and the icon.

In the player's client, the slow counters are displayed below the coins. One dark mushroom per counter.

The slow mechanic says : Reduce the player's movement for the turn by one. To a minimum of one.

So we have to add a boolean attribute to the player for his turn : has_already_moved_once.

If the player has not already moved once, he can always move one. If he has already moved once, the game checks if he has slow counters. If he does, the game removes one slow counter. If he doesn't, he can move.
